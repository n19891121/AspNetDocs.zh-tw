<span data-ttu-id="d05e0-123">MVC 模式可協助您建立應用程式，用來隔離應用程的不同層面 (輸入邏輯、商務邏輯和 UI 邏輯)，同時提供這些項目之間的鬆散結合。</span><span class="sxs-lookup"><span data-stu-id="d05e0-123">The MVC pattern helps you create apps that separate the different aspects of the app (input logic, business logic, and UI logic), while providing a loose coupling between these elements.</span></span> <span data-ttu-id="d05e0-124">此模式指定每一種邏輯應該位於應用程式中的位置。</span><span class="sxs-lookup"><span data-stu-id="d05e0-124">The pattern specifies where each kind of logic should be located in the app.</span></span> <span data-ttu-id="d05e0-125">UI 邏輯位於檢視。</span><span class="sxs-lookup"><span data-stu-id="d05e0-125">The UI logic belongs in the view.</span></span> <span data-ttu-id="d05e0-126">輸入邏輯位於控制器。</span><span class="sxs-lookup"><span data-stu-id="d05e0-126">Input logic belongs in the controller.</span></span> <span data-ttu-id="d05e0-127">商務邏輯則位於模型。</span><span class="sxs-lookup"><span data-stu-id="d05e0-127">Business logic belongs in the model.</span></span> <span data-ttu-id="d05e0-128">這項隔離可協助您管理建置應用程式時的複雜度，因為它可讓您一次處理實作的其中一個層面，而不影響另一個層面的程式碼。</span><span class="sxs-lookup"><span data-stu-id="d05e0-128">This separation helps you manage complexity when you build an app, because it enables you to work on one aspect of the implementation at a time without impacting the code of another.</span></span> <span data-ttu-id="d05e0-129">例如，您可以處理檢視程式碼，而不需要根據商務邏輯程式碼。</span><span class="sxs-lookup"><span data-stu-id="d05e0-129">For example, you can work on the view code without depending on the business logic code.</span></span>
