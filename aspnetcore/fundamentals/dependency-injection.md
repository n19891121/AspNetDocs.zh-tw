## <a name="overview-of-dependency-injection"></a><span data-ttu-id="490da-108">相依性插入概觀</span><span class="sxs-lookup"><span data-stu-id="490da-108">Overview of dependency injection</span></span>

<span data-ttu-id="490da-109">「相依性」是另一個物件所需的任何物件。</span><span class="sxs-lookup"><span data-stu-id="490da-109">A *dependency* is any object that another object requires.</span></span> <span data-ttu-id="490da-110">檢查具有 `WriteMessage` 方法的下列 `MyDependency` 物件，應用程式中的其他類別相依於此物件：</span><span class="sxs-lookup"><span data-stu-id="490da-110">Examine the following `MyDependency` class with a `WriteMessage` method that other classes in an app depend upon:</span></span>

```csharp
public class MyDependency
{
    public MyDependency()
    {
    }

    public Task WriteMessage(string message)
    {
        Console.WriteLine(
            $"MyDependency.WriteMessage called. Message: {message}");

        return Task.FromResult(0);
    }
}
```

::: moniker range=">= aspnetcore-2.1"

<span data-ttu-id="490da-111">您可以建立 `MyDependency` 類別的執行個體，讓 `WriteMessage` 方法可供類別使用。</span><span class="sxs-lookup"><span data-stu-id="490da-111">An instance of the `MyDependency` class can be created to make the `WriteMessage` method available to a class.</span></span> <span data-ttu-id="490da-112">`MyDependency` 類別是 `IndexModel` 類別的相依性：</span><span class="sxs-lookup"><span data-stu-id="490da-112">The `MyDependency` class is a dependency of the `IndexModel` class:</span></span>

```csharp
public class IndexModel : PageModel
{
    MyDependency _dependency = new MyDependency();

    public async Task OnGetAsync()
    {
        await _dependency.WriteMessage(
            "IndexModel.OnGetAsync created this message.");
    }
}
```

::: moniker-end

::: moniker range="<= aspnetcore-2.0"

<span data-ttu-id="490da-113">您可以建立 `MyDependency` 類別的執行個體，讓 `WriteMessage` 方法可供類別使用。</span><span class="sxs-lookup"><span data-stu-id="490da-113">An instance of the `MyDependency` class can be created to make the `WriteMessage` method available to a class.</span></span> <span data-ttu-id="490da-114">`MyDependency` 類別是 `HomeController` 類別的相依性：</span><span class="sxs-lookup"><span data-stu-id="490da-114">The `MyDependency` class is a dependency of the `HomeController` class:</span></span>

<span data-ttu-id="490da-115">該類別會建立 `MyDependency` 執行個體並直接相依於該執行個體。</span><span class="sxs-lookup"><span data-stu-id="490da-115">The class creates and directly depends on the `MyDependency` instance.</span></span> <span data-ttu-id="490da-116">程式碼相依性 (例如前一個範例) 有問題，因此基於下列原因應該避免使用：</span><span class="sxs-lookup"><span data-stu-id="490da-116">Code dependencies (such as the previous example) are problematic and should be avoided for the following reasons:</span></span>

* <span data-ttu-id="490da-117">若要將 `MyDependency` 取代為不同的實作，必須修改該類別。</span><span class="sxs-lookup"><span data-stu-id="490da-117">To replace `MyDependency` with a different implementation, the class must be modified.</span></span>
* <span data-ttu-id="490da-118">若 `MyDependency` 有相依性，那些相依性必須由該類別設定。</span><span class="sxs-lookup"><span data-stu-id="490da-118">If `MyDependency` has dependencies, they must be configured by the class.</span></span> <span data-ttu-id="490da-119">在具有多個相依於 `MyDependency` 之多個類別的大型專案中，設定程式碼在不同的應用程式之間會變得鬆散。</span><span class="sxs-lookup"><span data-stu-id="490da-119">In a large project with multiple classes depending on `MyDependency`, the configuration code becomes scattered across the app.</span></span>
* <span data-ttu-id="490da-120">此實作難以進行單元測試。</span><span class="sxs-lookup"><span data-stu-id="490da-120">This implementation is difficult to unit test.</span></span> <span data-ttu-id="490da-121">應用程式應該使用模擬 (Mock) 或虛設常式 (Stub) `MyDependency` 類別，這在使用此方法時無法使用。</span><span class="sxs-lookup"><span data-stu-id="490da-121">The app should use a mock or stub `MyDependency` class, which isn't possible with this approach.</span></span>

<span data-ttu-id="490da-122">相依性插入可透過下列方式解決這些問題：</span><span class="sxs-lookup"><span data-stu-id="490da-122">Dependency injection addresses these problems through:</span></span>

* <span data-ttu-id="490da-123">使用介面來將相依性資訊抽象化。</span><span class="sxs-lookup"><span data-stu-id="490da-123">The use of an interface to abstract the dependency implementation.</span></span>

### <a name="disposal-of-services"></a><span data-ttu-id="490da-300">處置服務</span><span class="sxs-lookup"><span data-stu-id="490da-300">Disposal of services</span></span>

<span data-ttu-id="490da-324">單一服務需要具備執行緒安全。</span><span class="sxs-lookup"><span data-stu-id="490da-324">Singleton services need to be thread safe.</span></span> <span data-ttu-id="490da-325">如果單一服務相依於暫時性服務，暫時性服務可能也需要具備執行緒安全，視它如何由單一項目使用。</span><span class="sxs-lookup"><span data-stu-id="490da-325">If a singleton service has a dependency on a transient service, the transient service may also need to be thread safe depending how it's used by the singleton.</span></span>

## <a name="recommendations"></a><span data-ttu-id="490da-328">建議</span><span class="sxs-lookup"><span data-stu-id="490da-328">Recommendations</span></span>

* <span data-ttu-id="490da-329">不支援以 `async/await` 與 `Task` 為基礎的服務解析。</span><span class="sxs-lookup"><span data-stu-id="490da-329">`async/await` and `Task` based service resolution is not supported.</span></span> <span data-ttu-id="490da-330">C# 不支援非同步建構函式，因此建議的模式是以同步方式解析服務後使用非同步方法。</span><span class="sxs-lookup"><span data-stu-id="490da-330">C# does not support asynchronous constructors, therefore the recommended pattern is to use asynchronous methods after synchronously resolving the service.</span></span>

* <span data-ttu-id="490da-331">避免直接在服務容器中儲存資料與設定。</span><span class="sxs-lookup"><span data-stu-id="490da-331">Avoid storing data and configuration directly in the service container.</span></span> <span data-ttu-id="490da-332">例如，使用者的購物車通常不應該新增至服務容器。</span><span class="sxs-lookup"><span data-stu-id="490da-332">For example, a user's shopping cart shouldn't typically be added to the service container.</span></span> <span data-ttu-id="490da-333">組態應該使用[選項模式](xref:fundamentals/configuration/options)。</span><span class="sxs-lookup"><span data-stu-id="490da-333">Configuration should use the [options pattern](xref:fundamentals/configuration/options).</span></span> <span data-ttu-id="490da-334">同樣地，請避免只存在以允許存取某個其他物件的「資料持有者」物件。</span><span class="sxs-lookup"><span data-stu-id="490da-334">Similarly, avoid "data holder" objects that only exist to allow access to some other object.</span></span> <span data-ttu-id="490da-335">最好是透過 DI 要求實際項目。</span><span class="sxs-lookup"><span data-stu-id="490da-335">It's better to request the actual item via DI.</span></span>

* <span data-ttu-id="490da-336">避免以靜態方式存取服務 (例如，以靜態方式設定 [IApplicationBuilder.ApplicationServices](/dotnet/api/microsoft.aspnetcore.builder.iapplicationbuilder.applicationservices) 型別以四處使用)。</span><span class="sxs-lookup"><span data-stu-id="490da-336">Avoid static access to services (for example, statically-typing [IApplicationBuilder.ApplicationServices](/dotnet/api/microsoft.aspnetcore.builder.iapplicationbuilder.applicationservices) for use elsewhere).</span></span>

* <span data-ttu-id="490da-337">避免使用「服務定位器模式」。</span><span class="sxs-lookup"><span data-stu-id="490da-337">Avoid using the *service locator pattern*.</span></span> <span data-ttu-id="490da-338">例如，當您可以改用 DI 時，請勿叫用 <xref:System.IServiceProvider.GetService*> 來取得服務執行個體。</span><span class="sxs-lookup"><span data-stu-id="490da-338">For example, don't invoke <xref:System.IServiceProvider.GetService*> to obtain a service instance when you can use DI instead.</span></span> <span data-ttu-id="490da-339">另一個要避免的服務定位器變化是插入在執行階段解析相依性的處理站。</span><span class="sxs-lookup"><span data-stu-id="490da-339">Another service locator variation to avoid is injecting a factory that resolves dependencies at runtime.</span></span> <span data-ttu-id="490da-340">這兩種做法都會混用[控制反轉](/dotnet/standard/modern-web-apps-azure-architecture/architectural-principles#dependency-inversion)策略。</span><span class="sxs-lookup"><span data-stu-id="490da-340">Both of these practices mix [Inversion of Control](/dotnet/standard/modern-web-apps-azure-architecture/architectural-principles#dependency-inversion) strategies.</span></span>

* <span data-ttu-id="490da-341">避免以靜態方式存取 `HttpContext` (例如 [IHttpContextAccessor.HttpContext](/dotnet/api/microsoft.aspnetcore.http.ihttpcontextaccessor.httpcontext))。</span><span class="sxs-lookup"><span data-stu-id="490da-341">Avoid static access to `HttpContext` (for example, [IHttpContextAccessor.HttpContext](/dotnet/api/microsoft.aspnetcore.http.ihttpcontextaccessor.httpcontext)).</span></span>

<span data-ttu-id="490da-342">就像所有的建議集，您可能會遇到需要忽略建議的情況。</span><span class="sxs-lookup"><span data-stu-id="490da-342">Like all sets of recommendations, you may encounter situations where ignoring a recommendation is required.</span></span> <span data-ttu-id="490da-343">例外狀況很少見&mdash;大部分是架構本身內的特殊案例。</span><span class="sxs-lookup"><span data-stu-id="490da-343">Exceptions are rare&mdash;mostly special cases within the framework itself.</span></span>
