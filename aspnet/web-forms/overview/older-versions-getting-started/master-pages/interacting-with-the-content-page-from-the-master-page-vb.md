### <a name="instigating-programmatic-interaction-via-an-event-and-event-handlers"></a><span data-ttu-id="ee508-118">透過事件和事件處理常式 Instigating 程式設計互動</span><span class="sxs-lookup"><span data-stu-id="ee508-118">Instigating Programmatic Interaction via an Event and Event Handlers</span></span>

<span data-ttu-id="ee508-119">從主版頁面叫用內容頁面功能比其他方式更具挑戰性。</span><span class="sxs-lookup"><span data-stu-id="ee508-119">Invoking content page functionality from a master page is more challenging than the other way around.</span></span> <span data-ttu-id="ee508-120">由於內容頁面具有單一主版頁面，因此在 instigating 內容頁面的程式設計互動時，我們知道有哪些公用方法和屬性是可處置的。</span><span class="sxs-lookup"><span data-stu-id="ee508-120">Because a content page has a single master page, when instigating the programmatic interaction from the content page we know what public methods and properties are at our disposal.</span></span> <span data-ttu-id="ee508-121">不過，主版頁面可以有許多不同的內容頁面，每個都有自己的一組屬性和方法。</span><span class="sxs-lookup"><span data-stu-id="ee508-121">A master page, however, can have many different content pages, each with its own set of properties and methods.</span></span> <span data-ttu-id="ee508-122">然後，我們可以如何在主版頁面中撰寫程式碼，以便在我們不知道要在執行時間前叫用哪個內容頁面時，在其內容頁面中執行某些動作？</span><span class="sxs-lookup"><span data-stu-id="ee508-122">How, then, can we write code in the master page to perform some action in its content page when we don't know what content page will be invoked until runtime?</span></span>

<span data-ttu-id="ee508-123">請考慮使用 ASP.NET 的 Web 控制項，例如 Button 控制項。</span><span class="sxs-lookup"><span data-stu-id="ee508-123">Consider an ASP.NET Web control, such as the Button control.</span></span> <span data-ttu-id="ee508-124">按鈕控制項可以出現在任意數目的 ASP.NET 網頁上，而且需要一種機制，讓它可以警示頁面已被按一下。</span><span class="sxs-lookup"><span data-stu-id="ee508-124">A Button control can appear on any number of ASP.NET pages and needs a mechanism by which it can alert the page that it has been clicked.</span></span> <span data-ttu-id="ee508-125">這是使用*事件*來完成的。</span><span class="sxs-lookup"><span data-stu-id="ee508-125">This is accomplished using *events*.</span></span> <span data-ttu-id="ee508-126">特別是，按鈕控制項會在按一下時引發其 `Click` 事件;包含按鈕的 [ASP.NET] 頁面可以選擇性地透過*事件處理常式*來回應該通知。</span><span class="sxs-lookup"><span data-stu-id="ee508-126">In particular, the Button control raises its `Click` event when it is clicked; the ASP.NET page that contains the Button can optionally respond to that notification via an *event handler*.</span></span>

<span data-ttu-id="ee508-127">這個相同的模式可在其內容頁面中使用主版頁面觸發程式功能：</span><span class="sxs-lookup"><span data-stu-id="ee508-127">This same pattern can be used to have a master page trigger functionality in its content pages:</span></span>

1. <span data-ttu-id="ee508-128">將事件新增至主版頁面。</span><span class="sxs-lookup"><span data-stu-id="ee508-128">Add an event to the master page.</span></span>
2. <span data-ttu-id="ee508-129">每當主版頁面需要與其內容頁面通訊時，引發事件。</span><span class="sxs-lookup"><span data-stu-id="ee508-129">Raise the event whenever the master page needs to communicate with its content page.</span></span> <span data-ttu-id="ee508-130">例如，如果主版頁面需要警示其 [內容] 頁面，而使用者已將價格加倍，則其事件會在價格加倍後立即引發。</span><span class="sxs-lookup"><span data-stu-id="ee508-130">For example, if the master page needs to alert its content page that the user has doubled the prices, its event would be raised immediately after the prices have been doubled.</span></span>
3. <span data-ttu-id="ee508-131">在需要採取一些動作的內容頁面中，建立事件處理常式。</span><span class="sxs-lookup"><span data-stu-id="ee508-131">Create an event handler in those content pages that need to take some action.</span></span>

<span data-ttu-id="ee508-132">本教學課程的其餘部分會執行簡介中所述的範例;也就是 [內容] 頁面，其中列出資料庫中的產品，以及包含按鈕控制項以將價格加倍的主版頁面。</span><span class="sxs-lookup"><span data-stu-id="ee508-132">This remainder of this tutorial implements the example outlined in the Introduction; namely, a content page that lists the products in the database and a master page that includes a Button control to double the prices.</span></span>
