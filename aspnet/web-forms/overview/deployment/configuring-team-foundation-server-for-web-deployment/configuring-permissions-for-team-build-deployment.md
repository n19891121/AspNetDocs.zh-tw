## <a name="task-overview"></a><span data-ttu-id="34520-110">工作概觀</span><span class="sxs-lookup"><span data-stu-id="34520-110">Task Overview</span></span>

<span data-ttu-id="34520-111">當您安裝 Team Foundation Server (TFS) 2010年組建服務時，您會指定要用來執行服務的身分識別。</span><span class="sxs-lookup"><span data-stu-id="34520-111">When you install the Team Foundation Server (TFS) 2010 build service, you specify the identity with which you want the service to run.</span></span> <span data-ttu-id="34520-112">根據預設，這是在 Network Service 帳戶。</span><span class="sxs-lookup"><span data-stu-id="34520-112">By default, this is the Network Service account.</span></span> <span data-ttu-id="34520-113">或者，您可以設定為使用網域帳戶來執行組建服務。</span><span class="sxs-lookup"><span data-stu-id="34520-113">Alternatively, you can configure the build service to run using a domain account.</span></span>

## <a name="configuring-web-server-permissions"></a><span data-ttu-id="34520-119">設定網頁伺服器權限</span><span class="sxs-lookup"><span data-stu-id="34520-119">Configuring Web Server Permissions</span></span>

<span data-ttu-id="34520-120">中所述[選擇 Web 部署的權限方法](../configuring-server-environments-for-web-deployment/choosing-the-right-approach-to-web-deployment.md)，有兩種主要的方法，如果您想要將 web 套件部署至遠端網頁伺服器，您可以使用：</span><span class="sxs-lookup"><span data-stu-id="34520-120">As described in [Choosing the Right Approach to Web Deployment](../configuring-server-environments-for-web-deployment/choosing-the-right-approach-to-web-deployment.md), there are two main approaches you can use if you want to deploy web packages to a remote web server:</span></span>

- <span data-ttu-id="34520-121">將從遠端位置的應用程式部署將目標設為*Web Deployment Agent Service* （也稱為遠端代理程式） 在目的地伺服器上。</span><span class="sxs-lookup"><span data-stu-id="34520-121">Deploy the application from a remote location by targeting the *Web Deployment Agent Service* (also known as the remote agent) on the destination server.</span></span>
- <span data-ttu-id="34520-122">將從遠端位置的應用程式部署將目標設為*Internet Information Services* (*IIS) Web 部署的處理常式*目的地伺服器上。</span><span class="sxs-lookup"><span data-stu-id="34520-122">Deploy the application from a remote location by targeting the *Internet Information Services* (*IIS) Web Deploy Handler* on the destination server.</span></span>

<span data-ttu-id="34520-123">遠端代理程式在此情況下有兩個主要限制：</span><span class="sxs-lookup"><span data-stu-id="34520-123">The remote agent has two key limitations in this case:</span></span>

- <span data-ttu-id="34520-124">遠端代理程式只支援 NTLM 驗證。</span><span class="sxs-lookup"><span data-stu-id="34520-124">The remote agent supports only NTLM authentication.</span></span> <span data-ttu-id="34520-125">換句話說，此部署必須使用組建服務身分識別&#x2014;您無法模擬另一個帳戶。</span><span class="sxs-lookup"><span data-stu-id="34520-125">In other words, the deployment must use the build service identity&#x2014;you can't impersonate another account.</span></span>

<span data-ttu-id="34520-127">放在一起，這些兩個的限制進行遠端代理程式的方法不想要自動化 Team Build 部署。</span><span class="sxs-lookup"><span data-stu-id="34520-127">Together, these two limitations make the remote agent approach undesirable for an automated Team Build deployment.</span></span> <span data-ttu-id="34520-128">若要使用這種方法，您必須進行組建服務帳戶在任何目標 web 伺服器上的系統管理員。</span><span class="sxs-lookup"><span data-stu-id="34520-128">To use this approach, you'd need to make the build service account an administrator on any target web servers.</span></span>

<span data-ttu-id="34520-129">相反地，Web 部署的處理常式方法會提供各種不同的優點：</span><span class="sxs-lookup"><span data-stu-id="34520-129">In contrast, the Web Deploy Handler approach offers various advantages:</span></span>

- <span data-ttu-id="34520-130">Web 部署處理常式支援基本驗證，透過 HTTPS，可讓您將替代帳戶的認證傳遞至 IIS Web Deployment Tool (Web Deploy)。</span><span class="sxs-lookup"><span data-stu-id="34520-130">The Web Deploy Handler supports basic authentication over HTTPS, which allows you to pass the credentials of an alternative account to the IIS Web Deployment Tool (Web Deploy).</span></span>
- <span data-ttu-id="34520-131">您可以設定目標 web 伺服器，以允許非系統管理員的使用者，將內容部署至特定的 IIS 網站，使用 Web 部署處理常式。</span><span class="sxs-lookup"><span data-stu-id="34520-131">You can configure target web servers to allow non-administrator users to deploy content to specific IIS websites using the Web Deploy Handler.</span></span>

<span data-ttu-id="34520-132">如此一來，最好是清楚地將目標設 Web 部署處理常式，當您從 Team Build web 套件部署自動化。</span><span class="sxs-lookup"><span data-stu-id="34520-132">As a result, it's clearly preferable to target the Web Deploy Handler when you automate web package deployment from Team Build.</span></span> <span data-ttu-id="34520-133">這是建議的程序：</span><span class="sxs-lookup"><span data-stu-id="34520-133">This is the recommended process:</span></span>

<span data-ttu-id="34520-137">在  [Contact Manager](../web-deployment-in-the-enterprise/the-contact-manager-solution.md)範例方案，指定驗證類型 (基本或 NTLM)，Web Deploy 認證，以及環境特定專案檔中的端點位址 （遠端代理程式或 Web 部署的處理常式）。</span><span class="sxs-lookup"><span data-stu-id="34520-137">In the [Contact Manager](../web-deployment-in-the-enterprise/the-contact-manager-solution.md) sample solution, you specify the authentication type (basic or NTLM), the Web Deploy credentials, and the endpoint address (remote agent or Web Deploy Handler) in the environment-specific project file.</span></span> <span data-ttu-id="34520-138">這些值用來編寫和執行專案檔時，執行 Web Deploy 命令。</span><span class="sxs-lookup"><span data-stu-id="34520-138">These values are used to formulate and run a Web Deploy command when the project file is executed.</span></span> <span data-ttu-id="34520-139">如需詳細資訊，請參閱 <<c0> [ 部署網頁套件](../web-deployment-in-the-enterprise/deploying-web-packages.md)。</span><span class="sxs-lookup"><span data-stu-id="34520-139">For more information, see [Deploying Web Packages](../web-deployment-in-the-enterprise/deploying-web-packages.md).</span></span>
