<span data-ttu-id="cb233-109">這些教學課程中的部署方法是以[瞭解專案](../web-deployment-in-the-enterprise/understanding-the-project-file.md)檔中所述的分割專案檔案方法為基礎，其中的組建程式是由兩個專案&#x2014;檔所控制，其中包含適用于每個目的地環境的組建指示，另一個包含環境特定的組建和部署設定。</span><span class="sxs-lookup"><span data-stu-id="cb233-109">The deployment method at the heart of these tutorials is based on the split project file approach described in [Understanding the Project File](../web-deployment-in-the-enterprise/understanding-the-project-file.md), in which the build process is controlled by two project files&#x2014;one containing build instructions that apply to every destination environment, and one containing environment-specific build and deployment settings.</span></span> <span data-ttu-id="cb233-110">在組建期間，環境特定的專案檔會合並到環境無關的專案檔中，以形成一組完整的組建指示。</span><span class="sxs-lookup"><span data-stu-id="cb233-110">At build time, the environment-specific project file is merged into the environment-agnostic project file to form a complete set of build instructions.</span></span>

## <a name="what-are-the-issues-when-you-deploy-a-membership-database"></a><span data-ttu-id="cb233-111">當您部署成員資格資料庫時，會發生什麼問題？</span><span class="sxs-lookup"><span data-stu-id="cb233-111">What Are the Issues When You Deploy a Membership Database?</span></span>

<span data-ttu-id="cb233-112">在大多數情況下，當您設計資料庫的部署策略時，您必須考慮的第一件事是您想要部署的資料。</span><span class="sxs-lookup"><span data-stu-id="cb233-112">In most cases, when you devise a deployment strategy for a database, the first thing you need to consider is what data you want to deploy.</span></span> <span data-ttu-id="cb233-113">在開發或測試環境中，您可能會想要部署使用者帳戶資料，以方便快速且輕鬆地進行測試。</span><span class="sxs-lookup"><span data-stu-id="cb233-113">In a development or test environment, you might want to deploy user account data to facilitate quick and easy testing.</span></span> <span data-ttu-id="cb233-114">在預備或生產環境中，您不太可能想要部署使用者帳戶資料。</span><span class="sxs-lookup"><span data-stu-id="cb233-114">In a staging or production environment, it's very unlikely that you'd want to deploy user account data.</span></span>

<span data-ttu-id="cb233-115">可惜的是，ASP.NET 成員資格資料庫會引進一些特定的挑戰，讓這種決策更為複雜：</span><span class="sxs-lookup"><span data-stu-id="cb233-115">Unfortunately, ASP.NET membership databases introduce some specific challenges that make this decision a lot more complex:</span></span>

- <span data-ttu-id="cb233-116">僅限架構的部署會讓成員資格資料庫處於無法運作的狀態。</span><span class="sxs-lookup"><span data-stu-id="cb233-116">A schema-only deployment will leave the membership database in a non-operational state.</span></span> <span data-ttu-id="cb233-117">這是因為成員資格資料庫包含了資料庫所需的一些設定資料（在**aspnet\_SchemaVersions**資料表中），以便能夠運作。</span><span class="sxs-lookup"><span data-stu-id="cb233-117">This is because the membership database includes some configuration data (in the **aspnet\_SchemaVersions** table) that the database requires in order to function.</span></span> <span data-ttu-id="cb233-118">因此，如果您執行成員資格資料庫的僅限架構部署以排除使用者帳戶資料，您將需要執行部署後腳本，以新增基本的設定資料。</span><span class="sxs-lookup"><span data-stu-id="cb233-118">As such, if you perform a schema-only deployment of your membership database in order to exclude user account data, you'll need to run a post-deployment script to add the essential configuration data.</span></span>
- <span data-ttu-id="cb233-119">根據您的成員資格資料庫的設定方式而定，成員資格提供者可能會使用電腦金鑰來加密密碼，並將其儲存在資料庫中。</span><span class="sxs-lookup"><span data-stu-id="cb233-119">Depending on how your membership database is configured, the membership provider may use the machine key to encrypt passwords and store them in the database.</span></span> <span data-ttu-id="cb233-120">在此情況下，您使用資料庫部署的任何使用者帳戶資料將會在目的地伺服器上變成無法使用。</span><span class="sxs-lookup"><span data-stu-id="cb233-120">In this case, any user account data you deploy with the database will become unusable on the destination server.</span></span> <span data-ttu-id="cb233-121">基於這個理由，部署使用者帳戶資料並不是支援的案例。</span><span class="sxs-lookup"><span data-stu-id="cb233-121">For this reason, deploying user account data is not a supported scenario.</span></span>

## <a name="choosing-a-membership-database-strategy"></a><span data-ttu-id="cb233-122">選擇成員資格資料庫策略</span><span class="sxs-lookup"><span data-stu-id="cb233-122">Choosing a Membership Database Strategy</span></span>

<span data-ttu-id="cb233-123">當您選擇如何在企業伺服器環境中布建成員資格資料庫時，請使用下列指導方針：</span><span class="sxs-lookup"><span data-stu-id="cb233-123">Use these guidelines when you choose how to provision a membership database in an enterprise server environment:</span></span>

- <span data-ttu-id="cb233-124">請盡可能不要部署成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="cb233-124">Wherever possible, do not deploy membership databases.</span></span> <span data-ttu-id="cb233-125">相反地，請在目標資料庫伺服器上手動建立成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="cb233-125">Instead, create the membership database manually on the target database server.</span></span> <span data-ttu-id="cb233-126">如果您尚未自訂您的成員資格資料庫架構，只要使用[ASP.NET SQL Server 註冊工具（aspnet\_regsql）](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx)，就可以在目的地的原位中建立一個新的。</span><span class="sxs-lookup"><span data-stu-id="cb233-126">If you haven't customized your membership database schema, you can simply create a new one in situ at the destination using the [ASP.NET SQL Server Registration Tool (aspnet\_regsql.exe)](https://msdn.microsoft.com/library/ms229862(v=vs.100).aspx).</span></span>
- <span data-ttu-id="cb233-127">如果您沒有選項可以部署成員資格資料庫&#x2014;，例如，如果您對資料庫架構&#x2014;進行了廣泛的修改，就應該執行成員資格資料庫的僅限架構部署，以排除使用者帳戶資料，然後執行部署後腳本來新增任何必要的設定資料。</span><span class="sxs-lookup"><span data-stu-id="cb233-127">If you have no option but to deploy a membership database&#x2014;for example, if you've made extensive modifications to the database schema&#x2014;you should perform a schema-only deployment of the membership database, to exclude user account data, and then run a post-deployment script to add any required configuration data.</span></span> <span data-ttu-id="cb233-128">您可以在[如何：部署 ASP.NET 成員資格資料庫而不包含使用者帳戶](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx)中找到這些方法的廣泛指引。</span><span class="sxs-lookup"><span data-stu-id="cb233-128">You can find broad guidance on these approaches in [How to: Deploy the ASP.NET Membership Database Without Including User Accounts](https://msdn.microsoft.com/library/ff361972(v=vs.100).aspx).</span></span>

<span data-ttu-id="cb233-129">請務必記住，您的*成員資格資料庫的架構可能相當靜態*。</span><span class="sxs-lookup"><span data-stu-id="cb233-129">It's important to remember that *the schema of your membership database is likely to be fairly static*.</span></span> <span data-ttu-id="cb233-130">即使您已自訂成員資格資料庫，也不太可能需要定期&#x2014;更新架構，而不會變更與 web 應用程式或資料庫專案中的程式碼相同的頻率。</span><span class="sxs-lookup"><span data-stu-id="cb233-130">Even if you've customized the membership database, it's unlikely that you'll need to update the schema on a regular basis&#x2014;it's not going to change with the same frequency as the code in a web application or a database project.</span></span> <span data-ttu-id="cb233-131">因此，您應該不需要在任何自動化或單一步驟的部署程式中包含成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="cb233-131">As such, you shouldn't need to include the membership database in any automated or single-step deployment processes.</span></span>

## <a name="using-vsdbcmd-to-update-a-membership-database-schema"></a><span data-ttu-id="cb233-132">使用 VSDBCMD 來更新成員資格資料庫架構</span><span class="sxs-lookup"><span data-stu-id="cb233-132">Using VSDBCMD to Update a Membership Database Schema</span></span>

<span data-ttu-id="cb233-133">如果您在第一次部署之後修改成員資格資料庫的結構，您可能不會想要使用 Internet Information Services （IIS） Web 部署工具（Web Deploy）重新部署資料庫。</span><span class="sxs-lookup"><span data-stu-id="cb233-133">If you modify the structure of your membership database after your first deployment, you may not want to use the Internet Information Services (IIS) Web Deployment Tool (Web Deploy) to redeploy the database.</span></span> <span data-ttu-id="cb233-134">Web Deploy 中的資料庫部署功能並不包含對目的地資料庫&#x2014;進行差異更新的功能，Web Deploy 必須卸載並重新建立資料庫。</span><span class="sxs-lookup"><span data-stu-id="cb233-134">The database deployment functionality in Web Deploy doesn't include the capability to make differential updates to a destination database&#x2014;instead, Web Deploy must drop and re-create the database.</span></span> <span data-ttu-id="cb233-135">這表示您會遺失任何現有的使用者帳戶資料，這通常不會在預備或生產環境中出現。</span><span class="sxs-lookup"><span data-stu-id="cb233-135">This means that you lose any existing user account data, which is typically undesirable in staging or production environments.</span></span>

<span data-ttu-id="cb233-136">替代方式是使用 VSDBCMD 公用程式來更新目的地資料庫的架構。</span><span class="sxs-lookup"><span data-stu-id="cb233-136">The alternative is to use the VSDBCMD utility to update the schema of your destination database.</span></span> <span data-ttu-id="cb233-137">VSDBCMD 包含兩個重要功能。</span><span class="sxs-lookup"><span data-stu-id="cb233-137">VSDBCMD includes two important capabilities.</span></span> <span data-ttu-id="cb233-138">首先，它可以將現有資料庫的架構匯入到 .dbschema 檔案中。</span><span class="sxs-lookup"><span data-stu-id="cb233-138">First, it can import the schema of an existing database into a .dbschema file.</span></span> <span data-ttu-id="cb233-139">其次，它可以將 .dbschema 檔案部署到現有的資料庫做為差異更新，這表示它只會進行必要的變更，讓目標資料庫保持在最新狀態，而且您不會遺失任何資料。</span><span class="sxs-lookup"><span data-stu-id="cb233-139">Second, it can deploy a .dbschema file to an existing database as a differential update, which means that it only makes the changes required to bring the target database up to date and you don't lose any data.</span></span>

<span data-ttu-id="cb233-140">您可以使用下列高階步驟來更新成員資格資料庫架構：</span><span class="sxs-lookup"><span data-stu-id="cb233-140">You can use these high-level steps to update a membership database schema:</span></span>

1. <span data-ttu-id="cb233-141">使用 [VSDBCMD 匯**入**] 動作，為您的來源成員資格資料庫產生 .dbschema 檔案。</span><span class="sxs-lookup"><span data-stu-id="cb233-141">Use the VSDBCMD **Import** action to generate a .dbschema file for your source membership database.</span></span> <span data-ttu-id="cb233-142">[如何：從命令提示字元匯入架構](https://msdn.microsoft.com/library/dd172135.aspx)中會描述此程式。</span><span class="sxs-lookup"><span data-stu-id="cb233-142">This procedure is described in [How to: Import a Schema from a Command Prompt](https://msdn.microsoft.com/library/dd172135.aspx).</span></span>
2. <span data-ttu-id="cb233-143">使用 [VSDBCMD**部署**] 動作，將 .dbschema 檔案部署到目的地成員資格資料庫。</span><span class="sxs-lookup"><span data-stu-id="cb233-143">Use the VSDBCMD **Deploy** action to deploy the .dbschema file to your destination membership database.</span></span> <span data-ttu-id="cb233-144">[VSDBCMD 的命令列參考中會描述此程式。EXE （部署和架構匯入）](https://msdn.microsoft.com/library/dd193283.aspx)。</span><span class="sxs-lookup"><span data-stu-id="cb233-144">This procedure is described in [Command-Line Reference for VSDBCMD.EXE (Deployment and Schema Import)](https://msdn.microsoft.com/library/dd193283.aspx).</span></span>
