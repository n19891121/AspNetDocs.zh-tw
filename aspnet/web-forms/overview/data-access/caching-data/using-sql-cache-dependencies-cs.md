## <a name="understanding-notification-and-polling"></a><span data-ttu-id="91a53-124">瞭解通知和輪詢</span><span class="sxs-lookup"><span data-stu-id="91a53-124">Understanding Notification and Polling</span></span>

<span data-ttu-id="91a53-125">有兩種技術可以用來判斷資料庫中的資料何時已修改：通知和輪詢。</span><span class="sxs-lookup"><span data-stu-id="91a53-125">There are two techniques that can be used to determine when the data in a database has been modified: notification and polling.</span></span> <span data-ttu-id="91a53-126">透過通知，當特定查詢的結果在上次執行查詢之後已經變更時，資料庫就會自動發出警示 ASP.NET 執行時間，此時會收回與查詢相關聯的快取專案。</span><span class="sxs-lookup"><span data-stu-id="91a53-126">With notification, the database automatically alerts the ASP.NET runtime when the results of a particular query have been changed since the query was last executed, at which point the cached items associated with the query are evicted.</span></span> <span data-ttu-id="91a53-127">使用輪詢時，資料庫伺服器會維護特定資料表上次更新時間的相關資訊。</span><span class="sxs-lookup"><span data-stu-id="91a53-127">With polling, the database server maintains information about when particular tables have last been updated.</span></span> <span data-ttu-id="91a53-128">ASP.NET 執行時間會定期輪詢資料庫，以檢查哪些資料表在輸入快取之後已經變更。</span><span class="sxs-lookup"><span data-stu-id="91a53-128">The ASP.NET runtime periodically polls the database to check what tables have changed since they were entered into the cache.</span></span> <span data-ttu-id="91a53-129">已修改資料的資料表會收回其相關聯的快取專案。</span><span class="sxs-lookup"><span data-stu-id="91a53-129">Those tables whose data has been modified have their associated cache items evicted.</span></span>

<span data-ttu-id="91a53-130">通知選項所需的安裝比輪詢少，而且更細微，因為它會追蹤查詢層級的變更，而不是在資料表層級。</span><span class="sxs-lookup"><span data-stu-id="91a53-130">The notification option requires less setup than polling and is more granular since it tracks changes at the query level rather than at the table level.</span></span> <span data-ttu-id="91a53-131">可惜的是，通知僅適用于 Microsoft SQL Server 2005 的完整版本（亦即，非 Express 版本）。</span><span class="sxs-lookup"><span data-stu-id="91a53-131">Unfortunately, notifications are only available in the full editions of Microsoft SQL Server 2005 (i.e., the non-Express editions).</span></span> <span data-ttu-id="91a53-132">不過，您可以使用 [輪詢] 選項，將所有版本的 Microsoft SQL Server 從7.0 到2005。</span><span class="sxs-lookup"><span data-stu-id="91a53-132">However, the polling option can be used for all versions of Microsoft SQL Server from 7.0 to 2005.</span></span> <span data-ttu-id="91a53-133">由於這些教學課程使用 SQL Server 2005 的 Express edition，因此我們將著重于設定和使用 [輪詢] 選項。</span><span class="sxs-lookup"><span data-stu-id="91a53-133">Since these tutorials use the Express edition of SQL Server 2005, we will focus on setting up and using the polling option.</span></span> <span data-ttu-id="91a53-134">如需 SQL Server 2005 s 通知功能的進一步資源，請參閱本教學課程結尾的進一步閱讀章節。</span><span class="sxs-lookup"><span data-stu-id="91a53-134">Consult the Further Reading section at the end of this tutorial for further resources on SQL Server 2005 s notification capabilities.</span></span>

<span data-ttu-id="91a53-135">使用輪詢時，資料庫必須設定為包含名為 `AspNet_SqlCacheTablesForChangeNotification` 的資料表，其中包含三個數據行： `tableName`、`notificationCreated`和 `changeId`。</span><span class="sxs-lookup"><span data-stu-id="91a53-135">With polling, the database must be configured to include a table named `AspNet_SqlCacheTablesForChangeNotification` that has three columns - `tableName`, `notificationCreated`, and `changeId`.</span></span> <span data-ttu-id="91a53-136">此資料表包含每個資料表的資料列，其中有可能需要在 web 應用程式的 SQL 快取相依性中使用的資料。</span><span class="sxs-lookup"><span data-stu-id="91a53-136">This table contains a row for each table that has data that might need to be used in a SQL cache dependency in the web application.</span></span> <span data-ttu-id="91a53-137">[`tableName`] 資料行會指定資料表的名稱，而 `notificationCreated` 則表示資料列加入至資料表的日期和時間。</span><span class="sxs-lookup"><span data-stu-id="91a53-137">The `tableName` column specifies the name of the table while `notificationCreated` indicates the date and time the row was added to the table.</span></span> <span data-ttu-id="91a53-138">`changeId` 資料行的類型是 `int`，其初始值為0。</span><span class="sxs-lookup"><span data-stu-id="91a53-138">The `changeId` column is of type `int` and has an initial value of 0.</span></span> <span data-ttu-id="91a53-139">其值會隨著資料表的每次修改而遞增。</span><span class="sxs-lookup"><span data-stu-id="91a53-139">Its value is incremented with each modification to the table.</span></span>

<span data-ttu-id="91a53-140">除了 `AspNet_SqlCacheTablesForChangeNotification` 資料表以外，資料庫也需要在可能出現在 SQL 快取相依性中的每個資料表上包含觸發程式。</span><span class="sxs-lookup"><span data-stu-id="91a53-140">In addition to the `AspNet_SqlCacheTablesForChangeNotification` table, the database also needs to include triggers on each of the tables that may appear in a SQL cache dependency.</span></span> <span data-ttu-id="91a53-141">每當插入、更新或刪除資料列，並在 `AspNet_SqlCacheTablesForChangeNotification`中遞增資料表的 `changeId` 值時，就會執行這些觸發程式。</span><span class="sxs-lookup"><span data-stu-id="91a53-141">These triggers are executed whenever a row is inserted, updated, or deleted and increment the table s `changeId` value in `AspNet_SqlCacheTablesForChangeNotification`.</span></span>

<span data-ttu-id="91a53-142">當使用 `SqlCacheDependency` 物件快取資料時，ASP.NET 執行時間會追蹤資料表的目前 `changeId`。</span><span class="sxs-lookup"><span data-stu-id="91a53-142">The ASP.NET runtime tracks the current `changeId` for a table when caching data using a `SqlCacheDependency` object.</span></span> <span data-ttu-id="91a53-143">系統會定期檢查資料庫，而且 `changeId` 與資料庫中的值不同的任何 `SqlCacheDependency` 物件都會被收回，因為不同的 `changeId` 值表示自快取資料以來已經變更了資料表。</span><span class="sxs-lookup"><span data-stu-id="91a53-143">The database is periodically checked and any `SqlCacheDependency` objects whose `changeId` differs from the value in the database are evicted since a differing `changeId` value indicates that there has been a change to the table since the data was cached.</span></span>
