- <span data-ttu-id="2ad84-176">默認情況下,應用程式任務(輸入邏輯、業務邏輯和 UI 邏輯)、可測試性和測試驅動開發 (TDD) 分離。</span><span class="sxs-lookup"><span data-stu-id="2ad84-176">Separation of application tasks (input logic, business logic, and UI logic), testability, and test-driven development (TDD) by default.</span></span> <span data-ttu-id="2ad84-177">MVC 架構中的所有核心合約都是以介面為主，而且可以使用模擬物件測試，模擬物件是模仿應用程式中實際物件之行為的模擬物件。</span><span class="sxs-lookup"><span data-stu-id="2ad84-177">All core contracts in the MVC framework are interface-based and can be tested by using mock objects, which are simulated objects that imitate the behavior of actual objects in the application.</span></span> <span data-ttu-id="2ad84-178">您可以對應用程式進行單元測試，而不需在 ASP.NET 程序中執行控制器，如此使單元測試迅速而靈活。</span><span class="sxs-lookup"><span data-stu-id="2ad84-178">You can unit-test the application without having to run the controllers in an ASP.NET process, which makes unit testing fast and flexible.</span></span> <span data-ttu-id="2ad84-179">您可以使用與 .NET Framework 相容的任何單元測試架構。</span><span class="sxs-lookup"><span data-stu-id="2ad84-179">You can use any unit-testing framework that is compatible with the .NET Framework.</span></span>
