### <a name="pessimistic-concurrency-locking"></a><span data-ttu-id="03175-123">封閉式並行存取 （鎖定）</span><span class="sxs-lookup"><span data-stu-id="03175-123">Pessimistic Concurrency (Locking)</span></span>

<span data-ttu-id="03175-124">若您的應用程式確實需要防止在並行案例下發生的意外資料遺失，其中一個方法便是使用資料庫鎖定。</span><span class="sxs-lookup"><span data-stu-id="03175-124">If your application does need to prevent accidental data loss in concurrency scenarios, one way to do that is to use database locks.</span></span> <span data-ttu-id="03175-125">這就叫做*封閉式並行存取*。</span><span class="sxs-lookup"><span data-stu-id="03175-125">This is called *pessimistic concurrency*.</span></span> <span data-ttu-id="03175-126">例如，在您從資料庫讀取一個資料列之前，您會要求唯讀鎖定或更新存取鎖定。</span><span class="sxs-lookup"><span data-stu-id="03175-126">For example, before you read a row from a database, you request a lock for read-only or for update access.</span></span> <span data-ttu-id="03175-127">若您鎖定了一個資料列以進行更新存取，其他使用者便無法為了唯讀或更新存取而鎖定該資料列，因為他們會取得一個正在進行變更之資料的複本。</span><span class="sxs-lookup"><span data-stu-id="03175-127">If you lock a row for update access, no other users are allowed to lock the row either for read-only or update access, because they would get a copy of data that's in the process of being changed.</span></span> <span data-ttu-id="03175-128">若您鎖定資料列以進行唯讀存取，其他使用者也可以為了唯讀存取將其鎖定，但無法進行更新。</span><span class="sxs-lookup"><span data-stu-id="03175-128">If you lock a row for read-only access, others can also lock it for read-only access but not for update.</span></span>

### <a name="detecting-concurrency-conflicts"></a><span data-ttu-id="03175-158">偵測並行衝突</span><span class="sxs-lookup"><span data-stu-id="03175-158">Detecting Concurrency Conflicts</span></span>
