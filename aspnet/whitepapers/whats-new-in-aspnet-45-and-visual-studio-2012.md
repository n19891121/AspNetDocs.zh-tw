## <a name="aspnet-core-runtime-and-framework"></a><span data-ttu-id="a7415-176">ASP.NET Core 執行階段和架構</span><span class="sxs-lookup"><span data-stu-id="a7415-176">ASP.NET Core Runtime and Framework</span></span>

### <a name="asynchronously-reading-and-writing-http-requests-and-responses"></a><span data-ttu-id="a7415-177">以非同步方式讀取和寫入 HTTP 要求和回應</span><span class="sxs-lookup"><span data-stu-id="a7415-177">Asynchronously Reading and Writing HTTP Requests and Responses</span></span>

<span data-ttu-id="a7415-178">ASP.NET 4 引進了能夠讀取 HTTP 要求實體做為資料流，使用*HttpRequest.GetBufferlessInputStream*方法。</span><span class="sxs-lookup"><span data-stu-id="a7415-178">ASP.NET 4 introduced the ability to read an HTTP request entity as a stream using the *HttpRequest.GetBufferlessInputStream* method.</span></span> <span data-ttu-id="a7415-179">這個方法會提供資料流存取要求的實體。</span><span class="sxs-lookup"><span data-stu-id="a7415-179">This method provided streaming access to the request entity.</span></span> <span data-ttu-id="a7415-180">不過，它以同步方式執行，其中繫結要求的持續期間的執行緒。</span><span class="sxs-lookup"><span data-stu-id="a7415-180">However, it executed synchronously, which tied up a thread for the duration of a request.</span></span>

<span data-ttu-id="a7415-181">ASP.NET 4.5 支援能夠讀取資料流上 HTTP 要求實體中，以非同步方式，以及能夠以非同步方式排清。</span><span class="sxs-lookup"><span data-stu-id="a7415-181">ASP.NET 4.5 supports the ability to read streams asynchronously on an HTTP request entity, and the ability to flush asynchronously.</span></span> <span data-ttu-id="a7415-182">ASP.NET 4.5 也提供雙重緩衝 HTTP 要求實體，可提供您更輕鬆整合使用下游的 HTTP 處理常式，例如.aspx 頁面處理常式和 ASP.NET MVC 控制站的能力。</span><span class="sxs-lookup"><span data-stu-id="a7415-182">ASP.NET 4.5 also gives you the ability to double-buffer an HTTP request entity, which provides easier integration with downstream HTTP handlers such as .aspx page handlers and ASP.NET MVC controllers.</span></span>

<a id="_Toc318097374"></a>
#### <a name="improvements-to-httprequest-handling"></a><span data-ttu-id="a7415-183">HttpRequest 處理的增強功能</span><span class="sxs-lookup"><span data-stu-id="a7415-183">Improvements to HttpRequest handling</span></span>

<span data-ttu-id="a7415-184">從 ASP.NET 4.5 所傳回的 Stream 參考*HttpRequest.GetBufferlessInputStream*支援同步和非同步讀取的方法。</span><span class="sxs-lookup"><span data-stu-id="a7415-184">The Stream reference returned by ASP.NET 4.5 from *HttpRequest.GetBufferlessInputStream* supports both synchronous and asynchronous read methods.</span></span> <span data-ttu-id="a7415-185">*Stream*從傳回的物件*GetBufferlessInputStream*現在實作將 BeginRead 與 EndRead 方法。</span><span class="sxs-lookup"><span data-stu-id="a7415-185">The *Stream* object returned from *GetBufferlessInputStream* now implements both the BeginRead and EndRead methods.</span></span> <span data-ttu-id="a7415-186">非同步*Stream*方法可讓您以非同步方式讀取要求實體中的區塊，而 ASP.NET 會釋放目前執行緒的非同步讀取迴圈的每個反覆項目之間。</span><span class="sxs-lookup"><span data-stu-id="a7415-186">The asynchronous *Stream* methods let you asynchronously read the request entity in chunks, while ASP.NET releases the current thread between each iteration of an asynchronous read loop.</span></span>

<span data-ttu-id="a7415-187">ASP.NET 4.5 也已新增方法以經過緩衝處理的方式讀取要求的實體：*HttpRequest.GetBufferedInputStream*.</span><span class="sxs-lookup"><span data-stu-id="a7415-187">ASP.NET 4.5 has also added a companion method for reading the request entity in a buffered way: *HttpRequest.GetBufferedInputStream*.</span></span> <span data-ttu-id="a7415-188">這個新的多載的運作方式類似*GetBufferlessInputStream*，支援同步和非同步讀取。</span><span class="sxs-lookup"><span data-stu-id="a7415-188">This new overload works like *GetBufferlessInputStream*, supporting both synchronous and asynchronous reads.</span></span> <span data-ttu-id="a7415-189">不過，它會讀取，如*GetBufferedInputStream*也將實體位元組複製到 ASP.NET 內部緩衝區，以便下游的模組和處理常式仍然可以存取要求的實體。</span><span class="sxs-lookup"><span data-stu-id="a7415-189">However, as it reads, *GetBufferedInputStream* also copies the entity bytes into ASP.NET internal buffers so that downstream modules and handlers can still access the request entity.</span></span> <span data-ttu-id="a7415-190">例如，如果某些上游管線中的程式碼已讀取要求實體使用*GetBufferedInputStream*，您仍然可以使用*HttpRequest.Form*或*HttpRequest.Files*.</span><span class="sxs-lookup"><span data-stu-id="a7415-190">For example, if some upstream code in the pipeline has already read the request entity using *GetBufferedInputStream*, you can still use *HttpRequest.Form* or *HttpRequest.Files*.</span></span> <span data-ttu-id="a7415-191">這可讓您執行的非同步處理的要求 （例如，資料流處理大型檔案上的傳至資料庫），但仍執行的.aspx 頁面和 ASP.NET MVC 控制站之後。</span><span class="sxs-lookup"><span data-stu-id="a7415-191">This lets you perform asynchronous processing on a request (for example, streaming a large file upload to a database), but still run .aspx pages and MVC ASP.NET controllers afterward.</span></span>
