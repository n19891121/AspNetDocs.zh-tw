## <a name="anti-csrf-protection"></a><span data-ttu-id="d81fe-297">防 CSRF 防護</span><span class="sxs-lookup"><span data-stu-id="d81fe-297">Anti-CSRF Protection</span></span>

<span data-ttu-id="d81fe-298">跨站台要求偽造 (CSRF) 攻擊會將惡意網站傳送要求給使用者目前登入的有弱點網站的位置。</span><span class="sxs-lookup"><span data-stu-id="d81fe-298">Cross-Site Request Forgery (CSRF) is an attack where a malicious site sends a request to a vulnerable site where the user is currently logged in.</span></span> <span data-ttu-id="d81fe-299">為了協助防止 CSRF 攻擊，使用 ASP.NET MVC*防偽語彙基元*，也稱為要求驗證權杖。</span><span class="sxs-lookup"><span data-stu-id="d81fe-299">To help prevent CSRF attacks, ASP.NET MVC uses *anti-forgery tokens*, also called request verification tokens.</span></span> <span data-ttu-id="d81fe-300">其概念是，伺服器會將隨機產生的語彙基元放入網頁。</span><span class="sxs-lookup"><span data-stu-id="d81fe-300">The idea is that the server puts a randomly generated token into a web page.</span></span> <span data-ttu-id="d81fe-301">當用戶端提交資料給伺服器時，必須在要求訊息中包含此值。</span><span class="sxs-lookup"><span data-stu-id="d81fe-301">When the client submits data to the server, it must include this value in the request message.</span></span>

<span data-ttu-id="d81fe-302">防偽語彙基元運作，因為惡意的頁面無法讀取使用者的權杖，因為同源原則。</span><span class="sxs-lookup"><span data-stu-id="d81fe-302">Anti-forgery tokens work because the malicious page cannot read the user's tokens, due to same-origin policies.</span></span> <span data-ttu-id="d81fe-303">（同源原則會防止存取彼此的內容的兩個不同站台上裝載的文件）。</span><span class="sxs-lookup"><span data-stu-id="d81fe-303">(Same-origin policies prevent documents hosted on two different sites from accessing each other's content.)</span></span>

<span data-ttu-id="d81fe-304">ASP.NET MVC 提供防偽語彙基元，內建支援，透過[AntiForgery](https://msdn.microsoft.com/library/system.web.helpers.antiforgery.aspx)類別和[[ValidateAntiForgeryToken]](https://msdn.microsoft.com/library/system.web.mvc.validateantiforgerytokenattribute.aspx)屬性。</span><span class="sxs-lookup"><span data-stu-id="d81fe-304">ASP.NET MVC provides built-in support for anti-forgery tokens, through the [AntiForgery](https://msdn.microsoft.com/library/system.web.helpers.antiforgery.aspx) class and the [[ValidateAntiForgeryToken]](https://msdn.microsoft.com/library/system.web.mvc.validateantiforgerytokenattribute.aspx) attribute.</span></span> <span data-ttu-id="d81fe-305">目前，這項功能不會建置到 Web API。</span><span class="sxs-lookup"><span data-stu-id="d81fe-305">Currently, this functionality is not built into Web API.</span></span> <span data-ttu-id="d81fe-306">不過，SPA 範本包含 Web API 的自訂實作。</span><span class="sxs-lookup"><span data-stu-id="d81fe-306">However, the SPA template includes a custom implementation for Web API.</span></span> <span data-ttu-id="d81fe-307">此程式碼定義於`ValidateHttpAntiForgeryTokenAttribute`解決方案的 [篩選器] 資料夾中的類別。</span><span class="sxs-lookup"><span data-stu-id="d81fe-307">This code is defined in the `ValidateHttpAntiForgeryTokenAttribute` class, which is located in the Filters folder of the solution.</span></span> <span data-ttu-id="d81fe-308">若要深入了解防 CSRF Web API 中，請參閱[防止跨網站要求偽造 （csrf） 等攻擊的攻擊](../../../web-api/overview/security/preventing-cross-site-request-forgery-csrf-attacks.md)。</span><span class="sxs-lookup"><span data-stu-id="d81fe-308">To learn more about anti-CSRF in Web API, see [Preventing Cross-Site Request Forgery (CSRF) Attacks](../../../web-api/overview/security/preventing-cross-site-request-forgery-csrf-attacks.md).</span></span>
