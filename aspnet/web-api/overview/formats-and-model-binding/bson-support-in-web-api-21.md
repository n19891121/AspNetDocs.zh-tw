<span data-ttu-id="e8dc7-107">[BSON](http://bsonspec.org/)是二進位序列化格式。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-107">[BSON](http://bsonspec.org/) is a binary serialization format.</span></span> <span data-ttu-id="e8dc7-108">"BSON"代表"二進位 JSON"，但 BSON 和 JSON 序列化非常不同。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-108">"BSON" stands for "Binary JSON", but BSON and JSON are serialized very differently.</span></span> <span data-ttu-id="e8dc7-109">BSON 是 「 JSON 類似 」，因為物件以類似 JSON 的名稱 / 值組。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-109">BSON is "JSON-like", because objects are represented as name-value pairs, similar to JSON.</span></span> <span data-ttu-id="e8dc7-110">不同於 JSON 中，數值資料類型會儲存為位元組，而不是字串</span><span class="sxs-lookup"><span data-stu-id="e8dc7-110">Unlike JSON, numeric data types are stored as bytes, not strings</span></span>

<span data-ttu-id="e8dc7-111">BSON 設計成輕量型、 容易掃描，且速度快，可以編碼/解碼。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-111">BSON was designed to be lightweight, easy to scan, and fast to encode/decode.</span></span>

- <span data-ttu-id="e8dc7-112">BSON 相當的大小為 JSON。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-112">BSON is comparable in size to JSON.</span></span> <span data-ttu-id="e8dc7-113">根據資料，BSON 承載可能小於或大於 JSON 承載。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-113">Depending on the data, a BSON payload may be smaller or larger than a JSON payload.</span></span> <span data-ttu-id="e8dc7-114">序列化二進位資料，例如影像檔，因為二進位資料不是 base64 編碼，會小於 JSON、 BSON。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-114">For serializing binary data, such as an image file, BSON is smaller than JSON, because the binary data is not base64-encoded.</span></span>
- <span data-ttu-id="e8dc7-115">BSON 文件很容易就能掃描，因為項目會加上 [長度] 欄位中，因此剖析器可以略過項目而不需要加以解碼。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-115">BSON documents are easy to scan because elements are prefixed with a length field, so a parser can skip elements without decoding them.</span></span>
- <span data-ttu-id="e8dc7-116">編碼和解碼會有效率，因為數值資料類型會儲存成數字，不是字串。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-116">Encoding and decoding are efficient, because numeric data types are stored as numbers, not strings.</span></span>

<span data-ttu-id="e8dc7-117">原生用戶端，例如.NET 用戶端應用程式，可以受益於使用 BSON 取代文字為基礎的格式，例如 JSON 或 XML。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-117">Native clients, such as .NET client apps, can benefit from using BSON in place of text-based formats such as JSON or XML.</span></span> <span data-ttu-id="e8dc7-118">瀏覽器用戶端，您可能要堅持使用 JSON，因為 JavaScript 可以直接將轉換的 JSON 承載。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-118">For browser clients, you will probably want to stick with JSON, because JavaScript can directly convert the JSON payload.</span></span>

<span data-ttu-id="e8dc7-119">幸運的是，Web API 會使用[內容交涉](content-negotiation.md)，因此您的 API 可以支援這兩種格式，並讓選擇的用戶端。</span><span class="sxs-lookup"><span data-stu-id="e8dc7-119">Fortunately, Web API uses [content negotiation](content-negotiation.md), so your API can support both formats and let the client choose.</span></span>
